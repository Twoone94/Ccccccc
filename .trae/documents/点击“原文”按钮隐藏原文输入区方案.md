## 目标
- 点击界面上的“原文”按钮（带图标与文字的小胶囊）后，隐藏原文输入区对应的<section>，保持输入内容不丢失。

## 实现思路
- 在页面组件内加入布尔状态`showOriginal`，默认`true`。
- 给“原文”按钮增加`onClick`事件：将`showOriginal`置为`false`（或切换为`!showOriginal`以实现显示/隐藏切换）。
- 原文输入区的<section>根据`showOriginal`动态添加`hidden`类；这样不会卸载DOM，用户已输入的内容不会丢失。
- 为可访问性加上`role="button"`（如保持`div`）或改为`button`，并设置`aria-controls`和`aria-expanded`。

## 文件位置
- `src/pages/CopywritingCreationInterface.tsx`
  - “原文”按钮文本：`<p className="pl-[4px] text-[12px] text-primary">原文</p>`（约304行）
  - 原文输入<section>：`<section className="flex flex-col rounded-[8px] bg-surface p-[16px] h-full min-h-0" ...>`（约359与375行；包含`placeholder="请输入原文..."`的那块为目标）

## 改动步骤
1. 在组件顶层引入状态：
   ```tsx
   const [showOriginal, setShowOriginal] = useState(true)
   ```
2. 给“原文”按钮外层可点击容器加点击事件（保持样式不变）：
   ```tsx
   <button
     type="button"
     className="inline-flex items-center rounded-[6px] p-[4px]"
     aria-controls="original-section"
     aria-expanded={showOriginal}
     onClick={() => setShowOriginal(false)}
   >
     <div className="inline-flex flex-col pt-[3px] pb-[3px]">
       <img src="/image/mi1liw3x-9d8bxrt.svg" alt="Source Text" className="w-[18px] h-[22px]" />
     </div>
     <p className="pl-[4px] text-[12px] text-primary">原文</p>
   </button>
   ```
   - 若需要“点击再显示”，改为`onClick={() => setShowOriginal(v => !v)}`。
3. 为目标<section>添加受控隐藏：
   ```tsx
   <section
     id="original-section"
     className={`flex flex-col rounded-[8px] bg-surface p-[16px] h-full min-h-0 ${showOriginal ? '' : 'hidden'}`}
     style={{ minWidth: MIN_W }}
   >
     {/* 原有内容保持不动，包括<textarea placeholder="请输入原文..." ... /> */}
   </section>
   ```
   - 如果页面有两个<section>结构，请仅对包含`placeholder="请输入原文..."`的输入区应用上述受控隐藏。

## 可选增强
- 若项目已有`cn`或`clsx`工具，可用它们合并类名；否则沿用模板字符串。
- 将按钮键盘可操作性完善（`Enter`/`Space`触发）。
- 如需隐藏后保留占位布局，可用`opacity-0 pointer-events-none`替代`hidden`，但通常`hidden`更简单直观。

## 验证点
- 点击按钮后输入区不可见，其他区域布局正常，刷新后初始仍为显示状态。
- 隐藏→再显示时，<textarea>中的已输入文本仍存在（选择切换模式时验证）。
